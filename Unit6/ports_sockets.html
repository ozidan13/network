UDP & Ports

السرعة ضد الضمان — والبوابات

Netflix بيستخدم UDP مش TCP لـ Streaming. الـ DNS بيستخدم UDP مش TCP للاستفسارات. ليه البروتوكول الأسرع في بعض الحالات أفضل من الأضمن؟

UDP — اتحكم في Reliability بنفسك

Connectionless, No Guarantee — Maximum Speed

UDP Header = 8 bytes فقط. Source Port، Dest Port، Length، Checksum. خلاص. مفيش Seq Numbers، مفيش ACK، مفيش Connection Setup. بيبعت وبيسيب.

TCP — 20+ bytes header
✅ Reliable — لا Packet ضائع
✅ Ordered — الترتيب مضمون
✅ Flow + Congestion Control
❌ Overhead كبير
❌ Connection Setup يأخذ وقت
📌 HTTP، SSH، FTP، Email
UDP — 8 bytes header
✅ سريع جداً (Low Latency)
✅ Multicast/Broadcast support
✅ Application تتحكم في الـ Logic
❌ لا Reliability
❌ لا Ordering
📌 DNS، Video Streaming، VoIP، Gaming
⚡ WHY STREAMING USES UDP

Netflix Frame تتأخر 50ms → مش مشكلة (Low Latency أهم). Frame تتأخر 2 ثانية عشان TCP يعيد إرساله → مشكلة كبيرة. Packet Loss في Video = Artifact صغير مش هيتلاحظ.

Port Numbers — البوابات

0–65535 — How One IP Can Run 100 Services

الـ IP بيحدد الجهاز. الـ Port بيحدد التطبيق على الجهاز. نفس الـ IP — مئات المنافذ في نفس الوقت.

0–1023: Well-Known 1024–49151: Registered 49152–65535: Dynamic

Socket = IP + Port

The 5-Tuple That Uniquely Identifies Every Connection

كل Connection محدد بـ 5-Tuple فريد:

Protocol:TCP Source IP:192.168.1.5 Source Port:52341 (ephemeral) Dest IP:142.250.185.78 (google.com) Dest Port:443 (HTTPS)
→ هذه المجموعة الخمسة تعرّف الـ Connection ده بشكل فريد على الشبكة كلها.
