Congestion Control — TCP Slow Start

AIMD: Additive Increase Multiplicative Decrease

TCP ما بيعرفش سعة الشبكة — بيكتشفها بالتدريج. الـ CWND (Congestion Window) بيبدأ بـ 1 MSS وبيتضاعف مع كل RTT حتى خسارة = Packet Loss = قلّص.

مراحل TCP Congestion Control

🚀

Slow Start

CWND يبدأ بـ 1 MSS ويتضاعف كل RTT (أسرع من اسمه!). يستمر حتى ssthresh.

📈

Congestion Avoidance

بعد ssthresh: +1 MSS كل RTT (خطي). أبطأ وأأمن.

💥

Packet Loss (Timeout)

ssthresh = CWND/2 → CWND = 1. يرجع للـ Slow Start.

⚡

Fast Retransmit

3 Duplicate ACKs = Packet lost → أعيد الإرسال فوراً بدون انتظار Timeout.
