المبدأ اللي بيحكم كل شيء

The Core Concept Behind All Network Communication

لما بتبعت طرد بالبريد، مش بتحط الحاجة في إيدك وترمي. بتحطها في علبة ← بتكتب عليها عنوان ← بتضيف ورقة شحن ← بتحطها في حقيبة التوصيل. كل مرحلة بتضيف طبقة معلومات.

الـ Encapsulation بالظبط نفس الفكرة — كل طبقة OSI بتضيف Header (وأحياناً Trailer) على البيانات قبل ما تبعتها للطبقة اللي تحتها.

📦 Encapsulation = عملية إضافة Headers على البيانات عند كل طبقة OSI أثناء الإرسال (من أعلى لأسفل).
📬 Decapsulation = عملية إزالة الـ Headers عند الاستقبال (من أسفل لأعلى).

المرسل والمستقبل

Sender Encapsulates ↔ Receiver Decapsulates

📱 المرسل — Encapsulation
Layer 7: بيعمل الـ Data (HTTP Request)
Layer 4: TCP Header (Port Numbers)
Layer 3: IP Header (Src/Dst IP)
Layer 2: Ethernet Header (MAC)
Layer 1: Bits بتتبعت على الكابل
📡
💻 المستقبل — Decapsulation
Layer 1: بيستقبل الـ Bits
Layer 2: بيشيل الـ Ethernet Header
Layer 3: بيشيل الـ IP Header
Layer 4: بيشيل الـ TCP Header
Layer 7: بيوصله الـ Data الأصلي

الفكرة الجوهرية: كل طبقة بتتكلم مع نظيرتها على الجهاز التاني. الـ Layer 3 بتاع موبايلك بيتكلم مباشرة مع الـ Layer 3 بتاع السيرفر — الطبقات التانية شايلة المعلومات ومش عارفاها.

ليه التغليف موجود أصلاً؟

Why Not Just Send Raw Data?

التوجيه الدقيق

IP Header فيه عنوان المصدر والوجهة. من غيره، الـ Router مش هيعرف يوجه الـ Packet لفين.

كشف الأخطاء

الـ FCS (Frame Check Sequence) في الـ Ethernet Frame بيتحقق إن البيانات وصلت سليمة. لو كان فيه خطأ — بيتعرفه.

استقلالية الطبقات

كل طبقة بتعمل شغلها من غير ما تعرف محتوى الطبقة اللي فوقيها. HTTP مش مهمته العناوين — IP مش مهمته المحتوى.

دعم عدة بروتوكولات

نفس الـ Frame ممكن يشيل TCP أو UDP، ونفس الـ Packet ممكن يمشي على Ethernet أو WiFi — بفضل الـ Headers.
