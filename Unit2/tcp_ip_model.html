لأن الـ TCP/IP أقدم من الـ OSI! الإنترنت بدأ على TCP/IP سنة 1983. OSI جه إمتداد نظري متأخر. زي ما تقول القواعد النحوية اتكتبت بعد ما الناس اتكلمت!

📌 القاعدة: OSI للفهم النظري والـ Troubleshooting. TCP/IP للتطبيق الفعلي. لازم تعرف الاتنين!

The 4 Layers of TCP/IP

🌐
Layer 4: Application
يعادل OSI Layers 5+6+7

في TCP/IP، الطبقات 5,6,7 بتاعة OSI اتدمجت في طبقة Application واحدة. المبرمجون بيتعاملوا معها مباشرة عبر:

HTTP/HTTPS DNS FTP SMTP SSH DHCP
🚛
Layer 3: Transport
يعادل OSI Layer 4

TCP — التسليم الموثوق: ضمان وصول البيانات بالترتيب الصح. للملفات والبريد الإلكتروني.
UDP — السريع: مفيش ضمان بس أسرع. للـ Gaming، VoIP، DNS.

🗺️
Layer 2: Internet
يعادل OSI Layer 3

بيتعامل مع الـ IP Addresses والـ Routing. البروتوكول الأساسي: IP — كل packet بياخد IP Address للمصدر والوجهة.

⚡
Layer 1: Network Access
يعادل OSI Layers 1+2

دمج الطبقتين الفيزيائية والـ Data Link من الـ OSI. بيتعامل مع الـ MAC Addresses والـ Ethernet والـ WiFi.

Why TCP/IP Won the "Protocol Wars"

جاء الأول

TCP/IP اتصمم وانتشر مع ARPANET قبل OSI — الـ Internet كله كان شغال عليه قبل ما OSI يكتمل.

Open Source

TCP/IP مجاني ومفتوح للكل. أي شركة تقدر تطبقه بدون تراخيص. OSI كان فيه بيروقراطية إدارية أكتر.

عملي وبسيط

4 طبقات أبسط من 7 — التطبيق العملي أسهل. الـ Developers بيفضلوا نموذج أبسط يشتغلوا بيه.
