أساسيات العناوين

كل جهاز في العالم عنده عنوان بيميّزه — زي الرقم القومي بس للأجهزة. هنفهم نظام العناوين من الأساس.

ليه العناوين ضرورية؟

The Address Problem — How Does Data Know Where to Go?

الإنترنت فيه 5+ مليار جهاز متصل. لما بتبعت رسالة WhatsApp — إزاي الشبكة تعرف الرسالة هتروح لموبايل صاحبك بالذات ومش لأي حد تاني؟

الإجابة: كل جهاز عنده عنوانين مختلفين — IP Address للتوجيه بين الشبكات، وMAC Address للتوصيل داخل الشبكة المحلية.

📮 القياس المفيد:
IP Address = عنوان بيتك (يعرّفك جغرافياً — تتغير لو اشتقلت)
MAC Address = رقمك القومي (مش بيتغير — محفوظ في الـ Hardware)

IPv4 Structure — 32 Bits, 4 Octets

🔒 Private IP — شبكة محلية (192.168.x.x). مش بيوصل للإنترنت مباشرة.

الـ IP Addresses مقسمة لنوعين أساسيين:

Public IP

عنوان فريد على مستوى الإنترنت كله. ISP بياخد منك فلوس عشانه.

e.g., 156.213.45.12

Private IP

داخل الشبكة المحلية بس. ممكن يتكرر في بيوت مختلفة.

192.168.x.x / 10.x.x.x

تاني 3 bytes = رقم تسلسلي فريد للجهاز نفسه.

IP ضد MAC — إمتى يستخدم كل واحد؟

Two Addresses, Two Jobs

IP Address

Layer Layer 3 — Network
Format 192.168.1.105
طبيعة العنوان منطقي — بيتغير حسب الشبكة
من بيحدده الـ Router أو الـ Admin
وظيفته التوجيه بين الشبكات المختلفة
مثال عنوان بيتك — بيتغير لو انتقلت

MAC Address

Layer Layer 2 — Data Link
Format AA:BB:CC:DD:EE:FF
طبيعة العنوان فيزيائي — محروق في الـ NIC
من بيحدده الشركة المصنعة
وظيفته التوصيل داخل نفس الشبكة المحلية
مثال رقمك القومي — مش بيتغير

💡 كيف بيشتغلوا مع بعض؟ لما بتبعت Data: الـ IP بيوجه الـ Packet خارج شبكتك للإنترنت. الـ MAC بيوصل الـ Frame جوا شبكتك المحلية للـ Router. الـ Router بيبدّل الـ MAC في كل خطوة، بس الـ IP بيفضل ثابت.
