تصميم الشبكات الاحترافية

الـ Network Engineer مش بس بيحل مشاكل — بيصمم أنظمة من الصفر. كيف تبني شبكة 500 موظف تشتغل 99.999% من الوقت؟ الإجابة في الـ Three-Tier Architecture.

Three-Tier Hierarchical Design

The gold standard for enterprise network architecture

Cisco طورت الـ Three-Tier Model في التسعينيات — ولسه الشركات الكبيرة بستخدمه لأنه بيفصل المسؤوليات بوضوح ويسهّل الـ Scalability والـ Troubleshooting.

🔥
CORE
Layer 1
Core Layer — العمود الفقري
المهمة: نقل البيانات بسرعة عالية بين الـ Distribution switches. مفيش filtering، مفيش ACL، مفيش routing complexities — بس سرعة.
الأجهزة: Cisco Nexus 9500، Catalyst 9600 — 100GbE ports
SLA: 99.999% = أقل من 5.26 دقيقة downtime سنياً
Redundancy: Minimum 2 core switches + Redundant links
⟵ 10GbE / 40GbE Uplinks ⟶
⚡
DIST.
Layer 2
Distribution Layer — طبقة الذكاء
المهمة: Routing بين VLANs، تطبيق ACLs وPolicies، Route Summarization. ده مكان القرارات.
الأجهزة: Layer-3 Switches — Catalyst 9300، 9400
Protocols: HSRP أو VRRP للـ Gateway Redundancy
تصميم: دايماً جوزين (Pairs) — Active/Standby
⟵ 1GbE / 10GbE Uplinks + EtherChannel ⟶
💻
ACCESS
Layer 3
Access Layer — نقطة الدخول
المهمة: توصيل الـ End Devices. VLAN Assignment، Port Security، PoE للـ Phones والـ APs.
الأجهزة: Catalyst 9200، 2960X — Layer-2 Switches
Security: DHCP Snooping، Dynamic ARP Inspection، 802.1X
نسبة: 1 Distribution switch لكل 4-8 Access switches
💡 Collapsed Core: للشبكات الصغيرة (أقل من 300 موظف) ممكن تدمج الـ Core والـ Distribution في نفس الأجهزة — اسمه Collapsed Core / Two-Tier Design. توفير في التكلفة مع الحفاظ على المبدأ.

Redundancy — مفيش Single Point of Failure

Design for failure — assume every link can go down

كل مكون ممكن يعطل — الكابل، الـ Switch، الـ PSU. المهندس الجيد بيصمم على فرضية إن أي حاجة ممكن تقع.

// Scenario: Primary link fails between Distribution and Core
Dist-SW1 ──10GbE──▶ Core-SW1 (PRIMARY — DOWN)
Dist-SW1 ──10GbE──▶ Core-SW2 (BACKUP — ACTIVE, failover <1s)
HSRP: Dist-SW2 becomes Active Gateway (STANDBY → ACTIVE)
EtherChannel
ربط 2-8 كابلات مع بعض زي لو كابل واحد. LACP (802.3ad). Bandwidth aggregation + Redundancy.
HSRP / VRRP
Default Gateway Redundancy. لو الـ Active Router وقع، الـ Standby فوراً بيأخد الـ Virtual IP.
Dual ISP
فرعين إنترنت من ISPs مختلفين. BGP للتوجيه بينهم. لو ISP1 وقع — ISP2 يشتغل.
STP / RSTP
يمنع الـ Loop في Redundant Links. RSTP أسرع من STP (convergence <1s vs 30-50s).

مبادئ التصميم الاحترافي

What separates a good network from a great one

📐
Scalability
صمم للضعف من أول. عندك 200 موظف دلوقتي؟ صمم لـ 400. الـ Three-Tier بيتوسع بسهولة — بس زود Access switches.
🔒
Segmentation
كل قسم في VLAN منفصل. لو في Breach في الـ Guest WiFi — مش بيوصل للـ Finance VLAN. عزل = حماية.
⚡
Simplicity
الـ Design الأبسط = الأسهل في الـ Troubleshooting. مش كل feature لازم تشغّلها. الـ Complexity = مشاكل.
📋
Documentation
Network Diagram، IP Plan، VLAN table، Change Log. بدون Documentation = شبكة ticking bomb. افتكر إنك ممكن تمرض وحد تاني يشتغل.
🔁
Redundancy
Dual Links، Dual Power، Dual ISP. كل single point of failure = كارثة محتملة. حساب الـ RTO والـ RPO من البداية.
📊
Manageability
SNMP، NetFlow، Syslog، NMS. تقدر تراقب كل switch وكل link من مكان واحد. Cisco DNA Center، SolarWinds، PRTG.
