User EXEC Mode — وضع القراءة فقط
للـ Basic monitoring فقط. ping، show version. مش تقدر تعدل أي إعداد. الدخول: تفتح Console أو SSH.
Router#
Privileged EXEC — وضع المشرف
الدخول: enable (أو en). كل الـ show commands، debug، copy، reload. مهم جداً: حماية بـ enable password.
Router(config)#
Global Configuration — وضع الإعداد
الدخول: configure terminal (أو conf t). هنا بتغير الـ hostname، بتعمل routing، بتضيف passwords. للخروج: exit أو Ctrl+Z.
Router(config-if)#
Interface Configuration — إعداد الـ Interface
الدخول: interface GigabitEthernet0/0. هنا بتحط IP Address، بتشغل/توقف الـ Interface، بتضبط Speed.
Router(config-router)#
Router Configuration — إعداد بروتوكول التوجيه
الدخول: router ospf 1 أو router rip. هنا بتضيف الشبكات، بتضبط Timer، بتعمل Redistribution.
💡 Shorthand: Cisco بتسمحلك تاخد أول حرفين/تلاتة من أي أمر — conf t بدل configure terminal، sh ip int br بدل show ip interface brief.

Lab عملي — إعداد Router من الصفر

Step-by-step router configuration walkthrough

📋 Scenario: Router جديد خارج من الكرتون. محتاج تعمل الإعدادات الأساسية وتوصله بـ Network.
1
تأمين الجهاز — Secure the Router
أول حاجة تعملها: ضيف Enable Password وامنع الوصول غير المصرح به.
Router# configure terminal
Router(config)# hostname R1
R1(config)# enable secret Str0ng@Pass! ! encrypted password
R1(config)# service password-encryption ! encrypt all passwords
R1(config)# no ip domain-lookup ! prevent typo DNS lookup delays
2
إعداد SSH — Enable Remote Access
Telnet مشفروش — SSH هو البديل الآمن للـ Remote Management.
R1(config)# ip domain-name company.local
R1(config)# crypto key generate rsa modulus 2048
R1(config)# ip ssh version 2
R1(config)# username admin privilege 15 secret Admin@2024
R1(config)# line vty 0 4
R1(config-line)# transport input ssh ! block telnet
R1(config-line)# login local
3
إعداد الـ Interfaces — Configure Interfaces
كل Interface محتاج IP، Description، وتفعيل بـ no shutdown.
R1(config)# interface GigabitEthernet0/0
R1(config-if)# description LAN-to-Switch-Core
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown
 
R1(config)# interface GigabitEthernet0/1 ! WAN interface
R1(config-if)# description WAN-to-ISP
R1(config-if)# ip address 203.0.113.2 255.255.255.252
R1(config-if)# no shutdown
4
Default Route — بوابة الإنترنت
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1 ! default route to ISP
5
التحقق والحفظ — Verify and Save
تحقق من الإعدادات قبل الحفظ — بعد الحفظ بتبقى دايمة حتى بعد الريستارت.
R1# show ip interface brief ! verify all interfaces UP
R1# show ip route ! verify routing table
R1# ping 203.0.113.1 ! test ISP gateway
R1# copy running-config startup-config ! or: write memory

اختبر نفسك — CCNA Style

These question types appear in the real CCNA 200-301 exam

أنت في Router# وكتبت configure terminal — إيه الـ Prompt اللي هيظهر؟

إيه الفرق بين enable password و enable secret؟

مفيش فرق — نفس الأمر
enable secret مشفر بـ MD5 في الكونفيج — enable password بيظهر كـ plain text
enable secret للـ SSH فقط
enable password أقوى لأنه أحدث

Interface GE0/0 عليها administratively down — إيه السبب وكيف تصلحها؟

الكابل مش موصول — غير الكابل
الـ IOS فيه bug — upgrade
تم تعطيلها يدوياً بـ shutdown — الحل: no shutdown في الـ Interface config mode
الـ VLAN مش موجود

عايز تحفظ الكونفيج عشان يبقى موجود بعد الريستارت — إيه الأمر الصح؟

show running-config
save config
copy running-config startup-config (أو write memory)
backup config flash
