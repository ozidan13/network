Service Models — مودلات الخدمة

What the cloud provider gives you — vs. what you manage yourself

🏗️
IaaS
Infrastructure as a Service
بتاخد VMs، Storage، Networks جاهزة — وإنت بتركّب عليها OS، Apps، كل حاجة.
أمثلة: AWS EC2، Azure VMs، GCP Compute Engine
مناسب لـ: Sysadmins، Network Engineers — عندك full control.
🛠️
PaaS
Platform as a Service
بتاخد platform جاهز — بس تحط الـ Code والـ Data. مش محتاج تدير OS أو Servers.
أمثلة: AWS Elastic Beanstalk، Azure App Service، Firebase
مناسب لـ: Developers — بيركزوا على الكود فقط.
📱
SaaS
Software as a Service
بتستخدم Software جاهز 100% — مش محتاج تعمل installation أو maintenance.
أمثلة: Gmail، Office 365، Salesforce، Zoom، Netflix
مناسب لـ: End Users — Zero IT skills needed.

AWS VPC — شبكتك في السحابة

Virtual Private Cloud — your isolated network on AWS

الـ VPC (Virtual Private Cloud) هو شبكتك الخاصة المعزولة على AWS. تخيله زي الـ LAN بتاعتك في الشركة — بس virtual وعلى AWS infrastructure.

🌐 AWS VPC — 10.0.0.0/16 (65,536 IPs)
🌍 Public Subnet — 10.0.1.0/24
🖥 Web Server⚖️ Load Balancer🔀 NAT Gateway
↳ Internet Gateway attached — Public IPs
🔒 Private Subnet — 10.0.2.0/24
🗄 Database RDS⚙️ App Servers🔑 Secrets Mgr
↳ No direct internet — NAT only (outbound)
🛡 Security Groups (stateful FW) 📋 NACLs (stateless FW) 🔀 Route Tables 🌐 Internet Gateway (IGW) 🔗 VPC Peering / Transit GW
IGWInternet Gateway — بوابة الإنترنت للـ Public Subnets. Attached للـ VPC واحدة.
NAT Gatewayبيسمح للـ Private Subnets تخرج للإنترنت (outbound فقط). Managed، highly available.
Security GroupFirewall على مستوى الـ Instance. Stateful — allow inbound يعني outbound تلقائي مسموح.
NACLNetwork ACL على مستوى الـ Subnet. Stateless — لازم تحدد inbound وoutbound منفصلين.
Route Tableبتحدد الـ traffic من كل Subnet بيروح فين — IGW، NAT، VPN، Peering.
VPC Peeringربط 2 VPCs مع بعض (نفس Region أو cross-region). Traffic بيسير على AWS backbone.
Transit GWHub-and-spoke لربط VPCs كتير + On-premises. Replaces complex peering meshes.

Hybrid Connectivity — ربط الـ On-Premises بالسحابة

Three ways to connect your office to AWS / Azure

🔐
Site-to-Site VPN
IPSec tunnel بين الـ On-Premises Router والـ AWS VGW (Virtual Private Gateway).
المزايا: سريعة في الإعداد، رخيصة (~$36/شهر)، مشفرة.
العيوب: تعتمد على الإنترنت — variable latency، محدودية الـ Bandwidth (1.25 Gbps).
مناسب لـ: Backup connectivity، development, startups.
⚡
AWS Direct Connect
خط مباشر dedicated بين الشركة وـ AWS — مش عبر الإنترنت العام.
المزايا: Consistent low latency، Bandwidth حتى 100 Gbps، مش بيتأثر بازدحام الإنترنت.
العيوب: تكلفة عالية (~$1000+/شهر)، إعداد أبطأ (weeks حتى months).
مناسب لـ: Banks، Healthcare، Enterprises — critical workloads.
🌐
SD-WAN (Software-Defined WAN)
بيجمع عدة links (MPLS + Internet + 4G) وبيشغلهم ذكياً. الـ Software بيقرر أي link يستخدم لكل نوع traffic.
المزايا: Zero-Touch Provisioning، Application-aware routing، ارخص من MPLS بـ 50-70%.
Vendors: Cisco Viptela، VMware Velocloud، Fortinet.
مناسب لـ: Multi-branch companies، Cloud-first enterprises.

اختبر نفسك

ليه الـ Private Subnet في AWS مش بيقدر يتصل بالإنترنت directly؟

لأن AWS بتمنع الـ Private Subnets من الإنترنت
لأن الـ Route Table بتاعه مش فيها route للـ Internet Gateway — و NAT Gateway بيسمح Outbound فقط
لأن الـ Security Groups بتحجبه
لأن AWS مش بتديه Public IP

شركة بنك محتاجة تشغّل Database على AWS بـ latency ثابتة وأمان عالي — إيه الأنسب؟

Site-to-Site VPN — عشان أرخص
Public Internet — أسرع
AWS Direct Connect — خط dedicated بدون إنترنت عام، latency ثابتة
SD-WAN — أحدث

إيه الفرق بين Security Group والـ NACL في AWS؟

مفيش فرق — نفس الحاجة
NACL أقوى دايماً
Security Group — Stateful على مستوى Instance. NACL — Stateless على مستوى Subnet (لازم تحدد inbound وoutbound منفصلين)
Security Group للـ Outbound فقط
