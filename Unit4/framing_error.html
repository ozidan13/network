كشف وتصحيح الأخطاء

كل ثانية تنتقل فيها بيانات عبر شبكة — بعض الـ Bits ممكن تتقلب. كيف الشبكة بتعرف وبتصلح؟

الأخطاء حقيقية — مش نادرة

Bit Errors Are Real — Caused by Noise, Interference, Hardware

كابل Ethernet عادي — BER (Bit Error Rate) ≈ 10⁻¹² (خطأ واحد في كل تريليون bit). ده بيبان قليل، بس لو بتبعت 1 Gbps → تريليون bit كل ثانية → خطأ واحد كل ثانية! بدون كشف أخطاء، البيانات ممكن تتعطل بصمت.

⚡ REAL CONSEQUENCE

بنك بيحول مليون جنيه. Bit واحد اتقلب في رقم الحساب → الفلوس وصلت حساب غلط. Error Detection ضروري — مش luxury.

Parity Bit — أبسط طريقة

Even Parity — Interactive Demo

اضغط على أي Bit لتغييره. الـ Parity Bit (أحمر) بيتحسب تلقائياً عشان عدد الـ 1s يبقى زوجي دائماً.

⚠️ محدودية Parity: بيكشف الأخطاء الفردية فقط. لو اتنين bits اتقلبوا — مش بيكتشفه. ولا بيصحح — بس بيكشف.

Checksum — بتجمع وبتقارن

Checksum — Used in IP, TCP, UDP Headers

المرسل بيجمع كل bytes البيانات ويبعت الناتج معاها. المستقبل بيعمل نفس الجمع. لو الأرقام مختلفة — فيه خطأ.

Data bytes:0xA3 0x2F 0x71 0xCC
Sum:0xA3 + 0x2F + 0x71 + 0xCC = 0x20F
Checksum:Ones complement: 0xFDF0 → sent with data
Receiver sum:Data + Checksum = all 1s → ✅ No Error
If corrupted:Sum ≠ all 1s → ❌ Error Detected → Discard

📌 الـ Checksum بيستخدمه TCP Header وUDP Header وIP Header. أبسط وأسرع من CRC بس أضعف.

CRC — الأقوى وأشيع

Cyclic Redundancy Check — Powers Ethernet FCS

المرسل بيقسّم البيانات على polynomial (رقم generator) ويبعت الباقي (Remainder) كـ CRC مع البيانات. المستقبل بيعمل نفس القسمة — لو الباقي صفر = لا أخطاء.

Data (bits):1011001
Generator:1101 (degree 3 → append 3 zeros)
Padded:1011001000
XOR step 1:1011 XOR 1101 = 0110
XOR step 2:1100 XOR 1101 = 0001
XOR step 3:0011 XOR 0000 = 011
CRC (FCS):011 → sent as trailer

⭐ CRC-32 (بيستخدمه Ethernet FCS) بيكتشف كل أخطاء الـ Single, Double, Odd bits وكل أخطاء الـ Burst حتى 32 bit.

كشف ≠ تصحيح

Detection vs Correction — Different Tools, Different Costs

🔍 Error Detection Only
✅ Parity, CRC, Checksum
✅ بسيط وسريع وقليل Overhead
⚡ لو خطأ → الـ Packet بيتحذف → إعادة إرسال (ARQ)
📌 Ethernet, TCP, UDP يستخدموه
🔧 Error Correction (FEC)
✅ Hamming Code, Reed-Solomon
✅ بيصحح الخطأ بدون إعادة إرسال
⚡ Overhead أكبر بكتير
📌 WiFi, Satellite, CDs/DVDs
💡 ليه مش دايماً FEC؟ لأن تصحيح الخطأ بيحتاج Redundancy bits أكتير بكتير. الـ Hamming Code بيضيف 50%+ overhead. في الشبكات السلكية السريعة — أرخص وأسرع إنك تعيد الـ Packet من إنك تصحح الـ Error.
