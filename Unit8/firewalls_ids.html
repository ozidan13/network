Firewalls & IDS/IPS

الجدران النارية وأنظمة الكشف

Firewall بيفحص كل Packet ويقرر: Permit أو Deny. IDS بيلاقي الهجمات. IPS بيلاقيها ويوقفها. الثلاثة لازم يشتغلوا مع بعض.

أنواع الـ Firewalls

Gen 1: Packet Filtering (Stateless)
بيشوف Header بس (IP, Port, Protocol). مفيش State. رخيص لكن سهل يُخدع.
Gen 2: Stateful Inspection
بيتتبع الـ Connection State. يعرف لو الـ Packet جزء من Session مسموح به أو لا. الأشيع.
Gen 3: NGFW (Next-Gen)
Application Awareness + User Identity + SSL Inspection + IPS Built-in. Palo Alto, Fortinet, Cisco FTD.

محاكي الـ ACL (Firewall Rules)

Watch Packets Being Filtered

الـ Rules بتُطبَّق بالترتيب — أول Rule يتطابق يُنفَّذ (First Match):
permit tcp any host 10.0.0.5 eq 443 # Allow HTTPS to web server
permit tcp any host 10.0.0.5 eq 80 # Allow HTTP to web server
permit tcp 10.0.1.0 0.0.0.255 any eq 22 # Allow SSH from admin subnet
deny tcp any any eq 23 # Block Telnet (unencrypted)
deny ip any any # Implicit deny all

IDS ضد IPS

Detect Only vs Detect AND Prevent

👁️ IDS — Intrusion Detection System
يحلل Traffic ويتعرف على Patterns
يُنبه Admin عند اكتشاف هجوم
لا يوقف الهجوم
Out-of-band — نسخة من Traffic
✅ لا يأثر على الـ Performance
🛡️ IPS — Intrusion Prevention System
يحلل ويوقف الهجوم في الوقت الفعلي
Inline — كل Traffic يمر خلاله
يعمل Drop/Reset/Block تلقائياً
⚠️ قد يحجب Traffic حقيقي (False Positive)
📌 Snort، Suricata، Cisco Firepower
