علم تشفير البيانات

من شيفرة قيصر قبل 2000 سنة — لـ AES-256 اللي بيشفر البيانات دلوقتي — التشفير هو العمود الفقري للأمن الرقمي.

Symmetric ضد Asymmetric

One Key vs Two Keys

🔑 Symmetric
نفس الـ Key للتشفير والفك
✅ سريع جداً — للـ Bulk Data
❌ كيف تبعت الـ Key؟ (Key Exchange Problem)
📌 AES-128، AES-256، ChaCha20
🔓 يُستخدم: بعد Handshake، المحادثة الفعلية
🔑🔓 Asymmetric
Public Key للتشفير، Private Key للفك
✅ Key Exchange مشكلوش — Public Key = عام
❌ بطيء جداً — للـ Bulk Data
📌 RSA-2048، RSA-4096، ECC، ElGamal
🔓 يُستخدم: TLS Handshake، Digital Sig.

Hash Functions

One-way — No Decrypt — Deterministic

Hash Function = fingerprint رقمية. أي تغيير طفيف في الـ Input = Hash مختلف تماماً. لا طريقة للعودة للـ Input من الـ Hash (One-way).

Input: "Hello"
SHA-256: 185f8db32921bd46d35...c6041
Input: "Hello!" ← changed 1 character
SHA-256: 334d016f755cd6dc58c...b4893 ← completely different!
