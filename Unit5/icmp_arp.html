ICMP & ARP

النداء والاستجواب في الشبكة

كل مرة بتعمل Ping — بتستخدم ICMP. وكل مرة موبايلك يتصل بـ Router — بيعمل ARP أول. البروتوكولان اللي ما بيتكلمش عنهم.

ARP — من عنده الـ IP ده؟

Address Resolution Protocol — IP → MAC Translation

الـ IP بيحدد الجهاز على مستوى الإنترنت. بس الـ Ethernet Frame بيحتاج MAC Address. الـ ARP هو اللي بيحوّل IP لـ MAC على الـ Local Network.

1.PC-A يعرف IP المطلوب: 192.168.1.10 — بس ما يعرفش MAC
2.🔊 ARP Broadcast: "من عنده 192.168.1.10؟ أنا AA:BB:CC:11:22:33"
3.كل الأجهزة على الشبكة استقبلت الـ Broadcast
4.PC-B صاحب 192.168.1.10 يرد: "أنا! MAC بتاعي: DD:EE:FF:44:55:66"
5.PC-A حفظ الـ MAC في ARP Cache ✅
ARP Cache Table (على PC-A):

ICMP — رسائل التحكم

Internet Control Message Protocol — The Diagnostic Layer

ICMP مش لنقل البيانات — لنقل رسائل التشخيص والتحكم. لو الـ Router ما قدرش يوصل Packet — بيرد برسالة ICMP بدل إنه يصمت.

Echo Request/Reply

هو الـ Ping. بتبعت Echo Request وبتستنى Echo Reply. RTT = Round Trip Time.

Time Exceeded (TTL=0)

الـ Traceroute بيستخدمه. كل Router بيقلل TTL بـ 1. لو وصل صفر → Time Exceeded → بيعرف المسار.

Destination Unreachable

الـ Router أو Host مش قادر يوصّل الـ Packet. كودات مختلفة: Network، Host، Port، Protocol Unreachable.

Redirect

الـ Router بيقول للـ Host: "مسار أحسن موجود — ابعت هناك."
