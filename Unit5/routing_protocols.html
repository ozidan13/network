Distance Vector ضد Link State

Two Fundamentally Different Approaches to Routing

📍 Distance Vector
كل Router يعرف المسافة بس (Hops/Metric)
بيبعت نسخة من Routing Table لجيرانه
❌ Convergence بطيء
❌ Routing Loops ممكنة
📌 RIP, EIGRP (Hybrid)
الأمثولة: "اسأل جارك الطريق"
🗺️ Link State
كل Router عنده خريطة كاملة للشبكة
بيبعت LSA (Link State Advertisement) للكل
✅ Convergence سريع جداً
✅ لا Routing Loops
📌 OSPF, IS-IS
الأمثولة: "عندك خريطة Google Maps"

BGP — بروتوكول الإنترنت

The Protocol That Runs the Internet — Between AS

الإنترنت مكونش من شبكة واحدة — ده مجموعة من Autonomous Systems (AS). كل AS (زي Vodafone مصر، Google، Amazon) له رقم ASN. BGP هو اللي بيتكلم بينهم.

# BGP Route Example
Network: 8.8.8.0/24
AS Path: 15169 (Google) ← 6762 (TelecomItalia) ← 8452 (TE)
Next Hop: 10.100.5.1
Local Pref: 100

⭐ BGP بياخد قرارات على أساس Policy (سياسة) مش بس المسافة. ممكن Vodafone تقرر تمشي Traffic بناءً على عقود تجارية — مش بس أقل Hops.
