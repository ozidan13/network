NAT & DHCP

سحر الـ IP الواحد وتوزيع العناوين

بيتك فيه 10 أجهزة — كلهم بيوصلوا الإنترنت بـ IP واحد بس. ده الـ NAT. وكل جهاز بياخد IP تلقائياً بدون ما تعمل أي حاجة — ده الـ DHCP.

أنواع الـ NAT

Static, Dynamic, PAT (Masquerade) — Three Flavors

Static NAT
IP داخلي واحد ↔ IP خارجي واحد (ثابت). للـ Servers اللي محتاجة IP ثابت من الخارج.
Dynamic NAT
Pool من IPs الخارجية — كل جهاز بياخد واحد عند الحاجة. لما ينتهي الـ Pool → Connection failed.
PAT (Port Address Translation) / NAT Overload
الأشيع! كل الأجهزة الداخلية بيشتركوا في IP خارجي واحد — مع Ports مختلفة للتمييز. الراوتر بيتتبع كل connection بـ (Inside IP:Port ↔ Outside IP:Port).

DHCP — عملية DORA

Discover → Offer → Request → Acknowledge

D
DISCOVER
العميل بيبعت Broadcast: "محتاج IP — في Server هنا؟" (Src: 0.0.0.0 → Dst: 255.255.255.255)
O
OFFER
الـ Server بيرد: "اتفضل IP: 192.168.1.10 — Lease: 24 ساعة — Gateway: 192.168.1.1"
R
REQUEST
العميل بيبعت Broadcast: "قبلت العرض من الـ Server ده" (عشان كمان يعلم باقي الـ Servers)
A
ACKNOWLEDGE
الـ Server: "مظبوط! الـ IP بتاعك لمدة 24h — DNS: 8.8.8.8، Subnet: 255.255.255.0" ✅
