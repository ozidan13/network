كل DevOps وSysAdmin بيستخدم SSH بشكل يومي. وكل deploy لـ ملفات على Server بيمر بـ SFTP أو SCP.

FTP — Active ضد Passive

Two Modes — One Broken Behind NAT

⚠️ Active FTP
Client: PORT 52000
Server connects TO Client:52000
❌ بيفشل وراء NAT/Firewall
❌ الـ Firewall بيبلوك Connection الجاي
📌 Port 21 (Control), 20 (Data)
✅ Passive FTP
Client: PASV
Server: "Connect me on port 55000"
✅ بيشتغل وراء NAT
✅ Client هو اللي بيفتح الـ Connection
📌 الأشيع — مستخدم في FTPS
⚠️ FTP بيبعت username وpassword وبيانات كلها Plain Text — استخدم FTPS (FTP + TLS) أو SFTP (SSH File Transfer Protocol) دايماً!

SSH — الباب الذكي

Secure Shell — Port 22 — Encrypted Everything

SSH بيوفر: Shell مشفرة + Port Forwarding + File Transfer (SCP/SFTP). كل حاجة مشفرة — حتى Username وPassword.

🔑 Key-Based Authentication (أأمن وأريح)
1. Generate key pair: ssh-keygen -t ed25519 -C "ahmed@laptop"
2. Copy public key to server: ssh-copy-id user@server
3. Connect without password: ssh user@server.com
ahmed@laptop:~$

SFTP ≠ FTPS — إيه الفرق؟

نفس الشيء — مجرد اسم مختلف
SFTP = FTP فوق SSH (Port 22). FTPS = FTP + TLS (Port 990/21)
