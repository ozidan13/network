DNS Deep Dive

دفتر هاتف الإنترنت

كل مرة بتكتب google.com — الكمبيوتر بيسأل "إيه الـ IP؟". هذا السؤال الصغير بيمر على 4 Servers في أقل من 100ms.

رحلة الـ DNS Query

From Browser to IP — 8 Steps in <100ms

كتبت www.google.com في المتصفح. الـ DNS Resolver بيعمل الرحلة دي:

1
يشوف في Browser Cache — هل حفظ الـ IP قبل كده؟ التحقق من DNS record TTL
2
OS Cache — يسأل الـ OS Cache (يشمل hosts file)
3
Recursive Resolver — يسأل DNS Server بتاع الـ ISP (أو 8.8.8.8 Google)
4
Root Server — ما يعرفش google.com بس يعرف من مسؤول عن .com
5
TLD Server (.com) — يعرف من مسؤول عن google.com
6
Authoritative NS لـ google.com — يرد بـ IP الفعلي: 142.250.185.78
7
الـ Resolver بيرجع الـ IP للـ Browser ويحفظه في Cache (لحد ما الـ TTL ينتهي)
8
المتصفح بيتصل بـ 142.250.185.78:443 ويفتح Google ✅

DNS Record Types

The Building Blocks of Every Domain

A
اسم → IPv4. الأساسي. google.com → 142.250.x.x
AAAA
اسم → IPv6. للقادم من الإنترنت.
CNAME
اسم → اسم آخر (alias). www → apex domain.
MX
Mail eXchange. يحدد Server بريد الدومين.
NS
Name Server. من هو الـ Authoritative DNS؟
TXT
نص حر. للـ SPF، DKIM، التحقق من الدومين.
SOA
Start of Authority. بيانات الـ Zone الأساسية.
PTR
Reverse DNS. IP → اسم. للـ Email verification.

هجمات الـ DNS

DNS Spoofing, Cache Poisoning, DoH

☠️

DNS Spoofing

المهاجم بيرد بـ IP مزيف قبل الـ Resolver الحقيقي → بتفتح موقع وهمي.

💉

Cache Poisoning

حشر IP مزيف في Cache الـ DNS Server → كل المستخدمين يُوجَّهون للموقع الوهمي.

🔒

DNSSEC

توقيع رقمي على DNS Records — المستقبل يتحقق من صحة الرد.

🛡️

DoH / DoT

DNS over HTTPS/TLS — الاستفسارات مشفرة. ISP ما يقدرش يشوف الدومينات.
