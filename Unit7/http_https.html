HTTPS — TLS Handshake

How HTTPS Establishes Encryption Before Data Transfer

1. Client Hello: Supported TLS versions, cipher suites, random bytes
2. Server Hello: Chosen cipher, server random + Certificate (public key)
3. Certificate verify + Key Exchange: Client verifies cert, generates Pre-Master Secret
4. Session Keys derived: Both derive identical session keys independently (ECDHE)
5. Encrypted Application Data ✅ AES-256-GCM / ChaCha20

💡 TLS 1.3 (الأحدث): 1-RTT Handshake (أسرع من TLS 1.2 الذي يأخذ 2-RTT). 0-RTT للاتصالات المتكررة. يُستخدم في HTTP/3.
