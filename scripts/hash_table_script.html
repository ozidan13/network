ما هى جداول التجزئة؟
قاموس الكمبيوتر: فكرة הـ Hash Table
🎥 [Medium Shot، بتشاور على الشاشة اللي فيها شكل القاموس] الـ Hash Table (أو Dictionary أو Map زي ما بتسمية بعض اللغات) هو باختصار شديد: دفتر عناوين.

عكس الـ Array اللي بيخزن الداتا في أدراج بناءً على الترتيب (الدرج رقم 0، الدرج 1، إلخ).. الـ Hash Table بيحفظ الداتا في أدراج عليها أسامي. بنسمي الاسم ده Key (المفتاح).. واللي جوة الدرج اسمه Value (القيمة).

Key-Value Pair: زي "Apple" سعرها "$1.2". أنت مش بتدور على تفاحة.. أنت بتروح لدرج الـ Apple تلاقيه مكتوب الفلوس.
لكن، المشكلة اللي بتواجه الكمبيوتر.. إن الذاكرة (Memory) مبتفهمش الحروف والكلمات (مبتفهمش يعنى إيه Apple). الذاكرة عبارة عن أرقام أدراج (0, 1, 2, 3..).
إزاي نخلي الكمبيوتر يربط كلمة "Apple" برقم درج معين عشان يروحه علطول؟
دالة التجزئة
الموظف السري العبقري: הـ Hash Function
🎥 [إضاءة بتميل للأزرق خفيف.. بتعمل حركة بصوابعك تحاكي التلقين] 🔊 [SFX: صوت آلة حاسبة سريعة جداً] هنا بيجي دور الساحر بتاعنا.. الدالة اللي اسمها Hash Function.

الـ Hash Function دي زي موظف الأرشيف العبقري. بتديلو كلمة معينة وليكن كلمة "Apple"... بيمسك الكلمة دي ويعمل عليها شوية عمليات رياضية ويحولها لرقم درُج... مثلاً: الدرج رقم 4.

بكرة لو سألته تاني: "هاتلي لو سمحت Apple".. هيעمل أوتوماتيك نفس الحسبة ويقولك "روح الدرج رقم 4 على طول!".

✨ [VFX: رسم توضيحي بيسحب كلمة "Apple" يدخلها في صندوق، وتخرج رقم 4] متخيل العظمة؟ مفيش لف ولا تدوير. إنت بتدخّله الكلمة، بيديلك العنوان بتاعها في הـ Memory. عملية الحسبة دي بتاخد O(1) Constants Time.. يعني بتخلص فوراً مهما كان حجم الداتا.
التصادمات
لما המوظف يغلط: مشكلة التصادمات!
🎥 [إضاءة بتميل للأحمر خفيف (Warning) مع زوم سريع] بس استنى.. مفيش موظف كامل! ماذا لو إدينا המوظف ده كلمة "Banana" وحولها برضه للدرج رقم 4؟ ودرج 4 ده أساساً إحنا شايلين فيه "Apple"؟

الكارثة دي اسمها في البرمجة: Hash Collision (التصادم). لما مفتاحين مختلفين الدالة بتاعتهم تديهم نفس العنوان!

🎥 [بتبص يمين وشمال كأنك بتدور على حل] المهندسين بيحلوا المشكلة دي بطريقتين أشهرهم اسمها Chaining. يعني كإنك بدل ما تحط الحاجة في الدرج وتقفله، الدرج ده بيبقى مفتوح وبيودي على قائمة (زي הـ Linked List اللي درسناها).
لو الدرج مليان.. اربطهم ببعض.

عيب الحل ده؟ إن لو فيه تصادمات كتير في الدرج ده بالذات، הـ Hash Table بتفقد سرعتها السحرية O(1) وتنحدر سرعتها وتبقى تبحث جوه القائمة واحد ورا التاني... O(n)!
معامل التحميل
إمتى נكسر المصنع ونبني جديد؟ (Load Factor)
🎥 [بترجع الكاميرا خطوة للوضع الهادي الواسع] عشان نتجنب הـ Collisions والكوابيس دي، لازم نفضل نراقب المخزن بتاعنا فيه كام درج مليان من الإجمالي. النسبة دي اسمها: Load Factor (معامل التحميل).

Load Factor = عدد العناصر الموجودة ÷ عدد أدراج הـ Array كلها.

يعني لو أدراجك 10، وانت حاطط فيهم 7 حاجات.. معامل التحميل بتاعك 0.7.

✨ [VFX: عداد زى عداد البنزين بيتملي للأحمر وتقوم الشاشة موسعة] في الـ Hash Tables، مجرد ما النسبة دي تقرب من 0.7، الكمبيوتر بيطلق الانذار، ويروح قالب הـ Memory وموسع المخزن (يضاعف الأدراج). عشان يدي براح ويبعد الداتا عن بعضها وميحصلش أي تصادم. عملية التوسعة دي بنسميها Resizing.
الخلاصة
دي الأقوى في الكون
🎥 [Medium Shot، بتبتسم وتقفل الكلام بثقة] الـ Hash Table هو الهيكل اللي كل لغة برمجة بتعتمده كعمود خيمتها (بيسموها Objects في جافاسكريبت، و Dict في بايثون).

أي حاجة محتاج تبحث عنها، تدخلها، تلغيها بسرعة قياسية غير مسبوقة O(1).. الـ Hash Tables بتخلص ده.

✨ [VFX: عنوان الدرس القادم "Graph Representation"] الآن وبعد ما فهمنا إزاى نحفظ הדاتا... تخيل الداتا دي بقت عبارة عن علاقات.. أصحاب على الفيس بوك! أنا صاحبك وصاحبك ده صاحب فلان... دي مش المصفوفات ومش الجداول اللي بتعبر عنها. دي عالم جديد تماماً اسمه الـ Graphs (الشيكات).
