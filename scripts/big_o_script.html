✨ [VFX: حرف O كبير بيظهر وفيه كود شغال جواه] هنا جي Big O Notation.

الـ Big O دي أداة رياضية بتجاوب على سؤال واحد بس: "لما حجم الداتا يعلى ويكبر جداً... الكود ده هيهنّج ويفطس في قد ايه؟"

Big O: بتوصف معدل النمو (Growth Rate) بتاع الكود. म्हणजे لما الديك Input أكبر... بكام ضعف الشغل بيزيد؟

وحرف الـ "n":
دايماً هتشوف حرف n.
n ده هو القلق كله. هو "حجم الداتا". لو عندك مصفوفة فيها ألف عنصر ← n = ألف. فيسبوك عنده 3 مليار مستخدم ← n = 3 مليار.
أسوأ احتمال مقدّر
ليه دايماً متقوقعين في الـ Worst Case؟
🎥 [بتبص للكاميرا بتشاؤم مصطنع] لما المبرمجين بييجوا يتكلموا عن الـ Big O... دايماً بيتكلموا عن أسوأ سيناريو.

ليه الحزن ده؟
تخيل أنا بدور على اسم "حسين" في لستة أسامي.
لو حسين ده أول اسم → دي اسمها Best Case. كود سريع ومفيش أحسن من كده.

بس في هندسة البرمجيات... إحنا مبنعملش حسابنا على الحظ. بنبني على إن الدنيا هتخبط فينا.
إحنا عايزين نعرف: لو حسين ده كان آخر اسم في اللستة، أو ملوش وجود أصلاً... السيرفر هياخد وقت أد ايه لحد ما يعيط؟

عشان كده الـ Big O بيركز دايماً على Worst Case (أسوأ حالة).

① الثائر الأسرع: O(1) - Constant Time ⚡

🎥 [بتتكلم باستمتاع] ده البرنس. الداتا 10 عناصر؟ بياخد ثانية. الداتا 10 مليار عنصر؟ بياخد ثانية. الوقت مش بيكبر أصلاً. ثابت.

② العبقري المكّار: O(log n) - Logarithmic Time 🚀

ده اللي اتكلمنا عنه الدرس اللي فات في Binary Search. ده اللي بيقلّل الشغل للنص مع كل حركة.

لو הדاتا كبرت 10 مليون ضعف... الوقت بتاعه هيزيد حاجات بسيطة جداً لا تذكر.
③ الموظف الملتزم: O(n) - Linear Time 📏

🎥 [بتعمل حركة موظف بيمضي حضور وانصراف] ده اللي بيمشي خطوة خطوة. الداتا زادت الضعف؟ هياخد وقت الضعف. علاقة طردية كلاسيكية.

التشبيه: راجل بيقرا كتاب صفحة صفحة.
// O(n) function search(arr, name) { for (let i = 0; i < arr.length; i++) { if(arr[i] === name) return true; } }
④ المدير الكُفء: O(n log n) 📊

ده أحسن ما يمكن الوصول ليه لما نيجي "نرتّب" حاجات (Sorting). بطيء شوية عن الـ O(n)، بس أحسن بمراحل من الكوارث اللي تحت.
أمثلة: Merge Sort, Quick Sort.
⑤ الغبي المجتهد: O(n²) - Quadratic Time 🐌

🎥 [بتمسك راسك كأنك مصدع] ده الكابوس الأول للمبرمجين. ده اللي بيقارن كل عنصر... بكل عنصر تاني. يعني لو عندك 100 عنصر، هيعمل 10,000 خطوة.
لو عندك ألف؟ مليون خطوة.

⑥ الوحش النووي: O(2ⁿ) - Exponential Time 💥

🎥 [بتقرب من الكاميرا، وبتهمس] لو كودك فيه التعقيد ده... أوعدك، إن لو الداتا كبرت لـ 50 عنصر بس... الكود هيفضل شغال لحد ما الشمس تطفى.
حرفياً، الرقم بيكبر لدرجة إن السنين مش هتكفي العمليات دي.
مثال: خوارزمية بتحل مشكلة بانها تجرب كل الاحتمالات الممكنة في الكون.

✨ [VFX: Rules popping up on screen dynamically] قاعدة 1: شفت Loop واحد بيلف على الداتا؟ → O(n).

قاعدة 2: شفت Loop جواه Loop؟ → O(n²).

قاعدة 3: شفت حاجة بتقسم الداتا للنص في كل مرة (زي Binary Search)؟ → O(log n).

قاعدة 4: الثوابت ملهاش لازمة:
يعني إيه؟ لو عندك Loop بيلف 100 مرة بس مش بيعتمد على n (مثلاً n زادت بس هو برضه هيلف 100 مرة) → ده O(1) مقنع!
طب لو عندك 2 لوب ورا بعض (مش جوه بعض)؟ يبقى O(2n)... بس في هندسة البرمجيات بنشيل الأرقام، فبيبقى برضة O(n).
