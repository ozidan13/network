الخطوات
4 خطوات — ده كل اللي محتاج تحفظه
🎥 [بتوقف وبتبص للكاميرا: "ركز معايا"] ✨ [VFX: 4 بوكسات بتظهر واحد واحد] الخوارزمية بتشتغل بـ 4 خطوات بس. أي حد يقدر يحفظهم:
① حط مؤشرين
واحد على أول عنصر — اسمه low (يبدأ من 0).
وواحد على آخر عنصر — اسمه high (يبدأ من length - 1).
دول بيرسموا "الحدود" بتاعة البحث. المنطقة اللي بندور فيها.
② احسب النص
mid = Math.floor((low + high) / 2)
بنجمع low + high ونقسم على 2. كده عرفنا العنصر اللي في "النص".
③ قارن وقرر
✅ لو arr[mid] == target ← لقيناه! خلاص 🎉
⬅️ لو arr[mid] > target ← الإجابة في الـ نص الشمال ← high = mid - 1
➡️ لو arr[mid] < target ← الإجابة في الـ نص اليمين ← low = mid + 1
④ كرر
ارجع للخطوة ② طالما low <= high.
لو low > high ← العنصر مش موجود.
🎥 [بتبص للكاميرا ببساطة] فاهم؟ اقسم. قارن. كرر. كده. مفيش أبسط من كده.

الكود
20 سطر كود... بس فيهم سحر
🎥 [بتقعد قدام اللابتوب وبتفتح الإيديتور] يلا نكتبها بالـ JavaScript. وهشرحلك كل سطر:
function binarySearch(arr, target) { let low = 0; // ① المؤشر على أول عنصر let high = arr.length - 1; // ① المؤشر على آخر عنصر while (low <= high) { // ④ كرر طالما فيه مكان تبحث فيه let mid = Math.floor((low + high) / 2); // ② احسب النص let guess = arr[mid]; // شوف إيه اللي في النص if (guess === target) { // ③ لو لقيناه... return mid; // رجّع الـ index بتاعه! } if (guess > target) { // ③ لو التخمين أكبر... high = mid - 1; // روح الشمال } else { // ③ لو التخمين أصغر... low = mid + 1; // روح اليمين } } return -1; // مش موجود! 😢 }

🎥 [بتبص للكاميرا] يلا نفهم كل سطر:

سطر 1: function binarySearch(arr, target) — بنعرّف الدالة. بتاخد حاجتين: المصفوفة المرتبة واللي بندور عليه.

سطر 2-3: بنحط المؤشرين — low على الأول و high على الآخر. دول حدود الملعب.

سطر 5: while (low <= high) — طالما الملعب لسه فيه مساحة نبحث فيها... نكمل!

سطر 6: بنحسب النص. Math.floor عشان لو طلع كسر (زي 4.5) ← ناخد 4.

سطر 9-10: لقيناه! الـ guess = الـ target. بنرجّع الـ index. مبروك!

سطر 12-13: التخمين أكبر ← الإجابة في الشمال ← بنقلص الملعب من فوق.

سطر 14-15: التخمين أصغر ← الإجابة في اليمين ← بنقلص الملعب من تحت.

سطر 18: خرجنا من اللوب ← العنصر مش موجود. بنرجّع -1.

يلا نجرب:
const arr = [2, 5, 8, 12, 16, 23, 38, 56, 72, 91]; console.log(binarySearch(arr, 23)); // 5 ← لقيناه! ✅ console.log(binarySearch(arr, 72)); // 8 ← لقيناه! ✅ console.log(binarySearch(arr, 50)); // -1 ← مش موجود ❌
5 8 -1
ليه سريع؟

log₂(n) = "كام مرة نقسم n على 2 لحد ما نوصل لـ 1"

والذاكرة؟ Space Complexity: O(1) — ثابتة. الخوارزمية بتحتاج 3 متغيرات بس: low، high، mid. مش بتعمل مصفوفات جديدة.
لو كتبت Binary Search بطريقة Recursive (الدالة بتنادي نفسها)، الـ Space Complexity هتبقى O(log n) بسبب الـ Call Stack. هنفهم ده في درس الـ Recursion.
