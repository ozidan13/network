فيلم Inception وحقيقة الحلم جوه الحلم
🎥 [بتبص للكاميرا وبتتكلم بصوت هادي، غموض خفيف] 🔊 [SFX: موسيقى غامضة شبه مزيكا Inception] شفتوا فيلم Inception بتاع كريستوفر نولان؟

ديكابريو كان بيدخل جوه حلم... وجوه الحلم ده بينام ويدخل حلم تاني... وجوه الحلم التاني ينام ويدخل حلم تالت.
ومشكلة الفيلم الكبيرة كانت: إزاي هيصحوا ويطلعوا بره كل ده؟

لو دخلوا أحلام للما لا نهاية مش هيصحوا أبداً! كان لازم يكون فيه نقطة معينة... زرار طوارئ، أو إشارة صحيان تخليهم يخلصوا الحلم التالت، فيصحوا في التاني، فيصحوا في الأول... ونرجع للواقع.

🎥 [بتبتسم وتقرب من الكاميرا] عالم البرمجة فيه الـ Inception بتاعه. فجأة... دالة (Function)، وهي شغالة... تنادي نفسها! فتدخل جوه نسخة تانية من نفسها... والنسخة التانية تنادي نفسها فتدخل في نسخة تالتة... وهكذا!

✨ [VFX: كلمة Recursion بتظهر مرسومة جوه نفسها وتصغر وتتكرر] الحوار ده اسمه Recursion (التكرار الذاتي).

لو معملتوش صح... كودك هيدخل في غيبوبة للما لا نهاية والبرنامج هينفجر. بس لو عملته بذكاء...؟ هتكتب بيه أعقد الخوارزميات في سطرين بس.

🔊 [SFX: دقة درامية BOOM] جاهز تدخل الحلم؟ يلا بينا.
المفهوم
يعني إيه Recursion؟ (تشبيه المرايا والبوكسات)
🎥 [بتمشي وتتكلم بإيدك] بعيداً عن الأفلام... تخيل صالون حلاقة. واقف بين مرايتين قصاد بعض. بتبص، بتشوف انعكاسك... وجوه الانعكاس فيه انعكاس أصغر... وجوه الأصغر فيه انعكاس أصغر لحد المالا نهاية.

Recursion: ببساطة، هو إنك تحل مشكلة كبيرة، عن طريق إنك تخلي الـ Function تنادي نفسها بس على جزء أصغر من المشكلة.

✨ [VFX: بوكس كبير بيتفتح، بيطلع منه بوكس أصغر متقفل، وتتكرر] مثال فظيع من كتاب Grokking Algorithms:
تخيل معاك كرتونة كبيرة مقفولة، ومطلوب منك تدور على مفتاح جواها.
تفتح الكرتونة تلاقي... بدل المفتاح، كراتين صغيرة!

بالبرمجة العادية هتعمل While Loop وتفكك كل الصناديق صندوق صندوق (وجع دماغ).

بالـ Recursion، بتعمل خطوتين طعمتين جداً:
1- طلع حاجة من الصندوق. لو لقيتها مفتاح؟ مبروك خلصنا!
2- لو لقيتها صندوق؟ نادي نفس الدالة تاني عشان تفتح الصندوق الجديد ده.

شفت الجمال؟ الدالة عالجت نفسها بنفسها!
القواعد الذهبية
إزاي متكسرش الكمبيوتر: قاعدة التوقف
🎥 [بتبص للكاميرا بتحذير وتوقف إيدك قدامك] 🔊 [SFX: صوت سحب فرامل] استنى! الدالة لو بتنادي نفسها... هتفضل تنادي نفسها للصبح والكمبيوتر هيهنج! عشان كده... كل دالة Recursion في الدنيا لازم يكون فيها جزئين:

1. الـ Base Case (حالة التوقف 🛑):
الشرط اللي بيقول للدالة: كفاية! ستوب! أنا لقيت المفتاح بلاش ننادي نفسنا تاني. من غير الـ Base Case... البرنامج هينفجر بحاجة اسمها Stack Overflow.
2. الـ Recursive Case (حالة التكرار 🔄):
اللحظة اللي الدالة بتنادي فيها نفسها على الـ input الأصغر.

🎥 [بتبتسم ابتسامة واثقة] يعني زي فيلم Inception: حالة التوقف (الـ Base case) هي الإشارة اللي بتصحيك من الحلم... قبل ما تغرق جوه للما لا نهاية.
الـ Call Stack
كومة الأطباق الجبارة (The Call Stack)
🎥 [في إيدك حزمة أطباق بلاستيك] طيب، لما دالة تنادى نفسها وهي لسه مخلصتش، إيه بيحصل في الذاكرة؟

الكمبيوتر عنده حاجة اسمها Call Stack (كومة النداءات). زي بالظبط لو مسكت الأطباق دي.

✨ [VFX: رسمة كومة أطباق بتترص فوق بعضها] لما الدالة الأولى تشتغل ← الكمبيوتر بيحط معلوماتها في طبق.
لما الدالة الأولى تنادي دالة ثانية (نسختها الأصغر) قبل ما تخلص ← بنحط طبق جديد فوق القديم.
لما الدالة الثانية تنادي ثالثة ← طبق ثالث فوق!

🎥 [بتشيل الطبق اللي فوق أوي] الكمبيوتر بيتعامل مع طبق واحد... الطبق اللي فوق خالص! أول ما الطبق اللي فوق خالص يخبط في الـ Base Case (شرط التوقف)، بيخلص شغلته... ونشيله من الكومة (Pop)! وننزل للطبق اللي تحته، نعطيه النتيجة عشان يكمل، ونشيله... لحد ما الكومة تفضي.
المثال الأول
المضروب (Factorial): 5! = 5 × 4!
🎥 [بتشاور على الشاشة، بتشرح بثقة] يلا نطبق! مثال الـ Factorial. (المضروب في الرياضيات).

إحنا عارفين إن 5! بيساوي: 5 × 4 × 3 × 2 × 1 = 120.

بس استنى! لاحظت حاجة؟
الجزئية بتاعة 4 × 3 × 2 × 1 دي... هي أصلاً 4!!
يعني 5! هي عبارة عن 5 × 4!.
وهي دي الـ Recursion! المشكلة مبنية على نسخة أصغر من نفسها!

function factorial(n) { // 🛑 1. Base Case (حالة التوقف عشان منجرّش للما لا نهاية!) if (n === 1) { return 1; } // 🔄 2. Recursive Case (نادي نفسك بس مع رقم أصغر) return n * factorial(n - 1); } console.log(factorial(5)); // 120


والذاكرة بتعمل إيه هنا؟ (الـ Stack):
- الطبق 1: factorial(5) بيقول "استنى، هضرب 5 في factorial(4) لما تيجي".
- الطبق 2: factorial(4) بيقول "استنى... هاتلي factorial(3)".
- الطبق 3: لحد ما نوصل لـ factorial(1).
✨ [VFX: الأطباق بتطير وتختفي بسرعة] أول ما نوصل لـ 1 (الـ Base Case)، يرجع النتيجة للّي تحته، ونضرب ونقفل الأطباق كلها!
الكابوس
فيبوناتشي O(2ⁿ): الكابوس اللي هيفجر اللابتوب
🎥 [النور بيغمق شوية، وصوتك بياخد شكل التحذير المرعب 😱] 🔊 [SFX: صوت درامي — دقات قلب] افتكروا معايا المتتالية الشهيرة Fibonacci اللي كل رقم فيها بيساوي مجموع الرقمين اللي قبله:
0, 1, 1, 2, 3, 5, 8, 13...

بما إن الرقم هو مجموع الرقمين اللي قبله، نقدر نكتبه بالـ Recursion:
// ⚠️ الكود ده كارثة نووية صامتة ❌ function fib(n) { if (n <= 1) return n; // 🛑 Base Case return fib(n - 1) + fib(n - 2); // 🔄 Recursive Case }

🎥 [بتبص في عين الكاميرا بالظبط] لو كتبت ده وشغلت fib(6)... هيشتغل.
أما لو طلبت fib(50)... اللابتوب بتاعك هيسخن لحد ما يسلق بيض، ومش هيطلع النتيجة!

ليه الكارثة دي؟!
لأنها O(2ⁿ) - Exponential Time (اللى حذرنا منه في درس الـ Big O).
لأن الدالة دي عشان تحسب fib(5) بتنادي fib(4) و fib(3).
و fib(4) بتنادي fib(3) و fib(2).
✨ [VFX: شجرة نداءات متفرعة بتظهر على الشاشة بتكبر بشكل مجنون] الكمبيوتر بيقعد يعيد حسابات fib(3) و fib(2) آلاف وملايين المرات من غير ما يحفظ النتيجة! كأنك بتسأل نفس السؤال لـ 10 طوابير وكلهم بيحسبوه من الصفر تاني!
المنقذ
سر الـ Memoization: احفظ عشان متتعبش
🎥 [بتمسح عرق وهمي وتقول بابتسامة] 🔊 [SFX: موسيقى النجاح] إزاي ننقذ الكود ده من الغباء؟
بسؤال واحد: "يا صاحبي، مش إحنا حسبنا fib(3) من شوية؟! بدل ما نحسبها تاني، تعالى نكتب النتيجة في ورقة ونسميها Cache أو Memo!"

هنا بتظهر تقنية عبقرية جداً اسمها Memoization.
// ✅ الكود السريع جداً O(n) function fibMemo(n, memo = {}) { // لو حسبناها قبل كده... هاتها من الورقة! if (memo[n]) return memo[n]; if (n <= 1) return n; // احفظ النتيجة في الورقة الأول قبل ما ترجعها memo[n] = fibMemo(n - 1, memo) + fibMemo(n - 2, memo); return memo[n]; }


✨ [VFX: صاروخ بينطلق] بمجرد ما ضفنا "ورقة الحفظ" دي... السرعة اتحولت من O(2ⁿ) المرعب... لـ O(n) الخطي السريع جداً!
fib(50) هتحسب النتيجة في الميكروثانية بدل السنين!
حرفيا... أنقذنا الكوكب. 🌍
