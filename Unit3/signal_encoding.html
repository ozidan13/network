المشكلة الأساسية

Digital → Analog → Digital — The Encoding Problem

الكمبيوتر يتكلم رقمياً: 0 و1. الكابل بيحمل إشارة فيزيائية: جهد كهربي، ضوء، أو موجة راديو. لازم تكون فيه قواعد اتفاق محددة للترجمة بينهم — وده اللي بيسموه Line Encoding.

لو قلنا "1 = +5V و0 = 0V" ببساطة — هيبقى فيه مشاكل كتير: إيمتى بدأت بيانات جديدة؟ إزاي أعرف لو فيه أصفار متتالية كتير؟ الساعة (Clock) بتتزامن ازاي؟

⚡ THE CORE CHALLENGE

لو بعتلك 10 أصفار متتالية كـ 0V — إزاي تفرق بين إنهم 10 صفر تاني عن بعض، ولا السلك وقف وبقى 0V طول الوقت؟ الإجابة هي الـ Encoding.

أنواع الـ Encoding

Line Encoding Schemes — Strengths & Weaknesses

NRZ

Non-Return to Zero. الأبسط — 1 = مستوى عالي، 0 = مستوى منخفض. المشكلة: أصفار أو واحدات كتير متتالية بتسبب مشكلة Synchronization.

NRZ-I

NRZ Inverted. عند الـ 1، الإشارة بتتعكس (invert). الـ 0 مش بيغير الإشارة. بيحل جزء من مشكلة التزامن للواحدات المتكررة.

Manchester

كل Bit فيه انتقال (transition) في المنتصف. 1 = انتقال لفوق، 0 = انتقال لتحت. بيحل مشكلة التزامن تماماً. بيستخدمه Ethernet 10Base-T.

AMI

Alternate Mark Inversion. الـ 0 = 0V، الـ 1 = يتبادل بين +V و-V. بيوفر إشارة متعادلة وكشف أخطاء مضمّن. بيستخدمه T1/E1 telephone lines.

4B/5B

كل 4 bits بتتحول لـ 5 bits عشان تضمن عدم وجود أكتر من 3 أصفار متتالية. بيستخدمه Fast Ethernet (100BASE-TX). 20% overhead لكن مضمون.

8B/10B

كل 8 bits بتتحول لـ 10 bits. Balance كهربي مضمون. بيستخدمه Gigabit Ethernet (1000BASE-T) وFiber Channel.

Baud Rate ≠ Bit Rate

Common Confusion — Cleared Once and For All

Baud Rate = عدد التغيرات في الإشارة في الثانية (Signal Changes/sec)
Bit Rate = عدد الـ Bits المنقولة في الثانية

لو كل تغيير في الإشارة بيمثّل أكتر من Bit واحد — الـ Bit Rate هيبقى أعلى من الـ Baud Rate!

Example: QAM-16 → each symbol = 4 bits

Baud Rate = 1,000 symbols/sec

Bit Rate = 1,000 × 4 = 4,000 bps

هذه هي الطريقة اللي WiFi وADSL بيحققوا بها سرعات عالية على نفس الـ Bandwidth — بزيادة عدد الـ Bits per Symbol.
